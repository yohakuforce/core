## v0.3.0 ✅ (実証完了)

**北極星**: 現参画プロジェクトで「資料更新が不要・ソースが正本」を週次運用で実証する。

**DoD (5 項目中 3 項目以上)**:
1. `yohaku sync` を 4 週連続で週次運用
2. 利用者主観「資料更新が不要」5 段階で 4 以上
3. 運用タスク AI 任せ可能比率 60% 以上
4. 1 機能あたりリリース準備時間 50% 短縮
5. 手動作業見逃し件数 0 / リリース

**制約**: 新機能追加なし、実利用と計測に集中。

## v0.5.0 (公開準備中) — HTML 設計書パイプライン

**北極星**: 同じ知識グラフから「AI が読む Markdown」と「人間がレビューする HTML 設計書」を二系統で生成し、レビュー・充填・プレビュー・リリース判断を一気通貫にする。

**主要トピック** (詳細は [CHANGELOG](../CHANGELOG.md) §0.5.0):
- HTML 設計書パイプライン (5 タイプ component leaf + JSON ドリブンホーム + Mermaid/HTML フォールバック)
- LLM ブロック充填 (`explain-prompts` / `html-write`)、AI-managed ブロック preservation
- ローカル開発サーバ (`serve --watch`、SSE 自動 reload)、Cmd+K グローバル検索
- リリースレビュー HTML (`diff --format html`)、テストカバレッジ統合 (`coverage import`)
- 業務ドメイン階層 (`domains`)、業務フロー俯瞰タブ

## 設計原則 (変更しない)

- **3 層分離**: 決定的処理 / AI 判断 / 人手補完を混ぜない
- **1 バージョン = 1 North Star**: 隣のバージョンに目移りしない
- **正本は実装側**: Markdown は派生物
- **AI に生データを読ませない**: 知識グラフ経由
