# v0.4.1 Release Notes — Docs sanitize & npm-first quickstart

> Release date: 2026-05-19
> GitHub Release 作成時にこのファイルの内容を使用する (メンテナーが実行)

---

## 概要

v0.4.1 は **機能変更なしのドキュメント整備リリース**です。

v0.4.0 で `@yohakuforce/core` が npm に publish された一方、README / quickstart は依然として `git clone` 起点のセットアップ手順を案内していました。本リリースで「OSS として配布されたパッケージを `npm install -g` でそのまま使える」状態に手順を揃えます。あわせて公開ドキュメント・ADR から内部情報 (個人名 / ローカルパス / 旧 GitHub org 名) を取り除き、`docs/rollout/` を **自組織導入用の汎用テンプレート集** として再構築しました。

配布物 (`dist`) には変更ありません。**コードを書いている人にとってはアップデート不要**です。

---

## What's New (Highlights)

### Quickstart が npm 起点になった

```bash
npm install -g @yohakuforce/core
yohaku --version

cd /path/to/your-salesforce-project
yohaku init --bootstrap --profile minimal
```

または `npx` で 1 回だけ試す:

```bash
npx -p @yohakuforce/core yohaku init --bootstrap --profile minimal
```

ソースからのビルドはコントリビュータ向けとして `CONTRIBUTING.md` に集約しました。

### `docs/rollout/` が自組織導入用テンプレート集になった

社内・チームに展開したい人が、自分の文脈に合わせて編集する前提のたたき台として再構築しました。

- 概要資料 (5 分で読める分量)
- Windows Quickstart
- 10〜15 分ハンズオン台本
- 想定 FAQ
- アナウンス文 (Slack / メール / 件名候補)

`<実施日>` `<発表者名>` `<チーム名>` 等をプレースホルダー化済み。`docs/rollout/README.md` に使い方を記載しています。

### 公開ドキュメントの sanitize

公開リポジトリ上のドキュメント・ADR から個人名 / 個人パス / 旧 GitHub org 名 (`pro-koya/yohakuforce`) を除去。ADR の `koya` 言及は `メンテナー` / `オーナー` に置換しています。

## Upgrade

```bash
npm update -g @yohakuforce/core
yohaku --version
# 0.4.1 が表示されれば OK
```

既存プロジェクトの再 `yohaku init` は不要です。

## Known Limitations (継続)

v0.4.0 から引き継ぎ:

- `method-summary-table` がインライン SOQL `[SELECT ...]` を検出しない
- `yohaku metrics` が Claude Code セッションのトークンを計測していない
- `/yohaku-explain` の一括実行モード (`--all` / カテゴリ単位) 未実装
