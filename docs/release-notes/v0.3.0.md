Week 0 (2026-05-11〜05-13) に発見・修正した既知の pitfalls を集約し、

## What's Changed

### Security Fixes (v0.2.1)

12 件のセキュリティ修正を実施:

- **CRITICAL**: `yohaku graph query` の任意 SQL 実行 — allowlist + readonly モードの二重防御
- **HIGH × 5**: シークレットマスキング / ReDoS / パストラバーサル (2 件) / git ref インジェクション
- **MEDIUM × 5**: fast-xml-parser 更新 (CVE 解消) / AI 出力サニタイズ / シェル FQN 検証 / symlink 制限 / gitignore 追加
- **推奨**: XMLParser に `processEntities: false` を明示

### Bug Fixes (v0.2.2)

- **Windows パス区切り互換** 5 箇所対応 — `profile` フィルタ / `--project-name` / archive / diff-classify が Windows でも正常動作
- **スキーマ検証エラーの可読性改善** — `instancePath` と内容を最大 5 件表示

### Schema Enhancements (v0.2.3)

- **Salesforce 公式 enum 値を拡充** — `sharingModel`, `EntityRef.kind`, `SharingRule.kind`, `AuraBundle.bundleKind` に公式値を追加
- **`.github/` PR / Issue テンプレート追加** — 第三者コントリビューターの参加経路を整備

### Mermaid Fix (v0.2.4)

- **Trigger / Flow フローチャートの矢印方向修正** — `Trigger → Object` を `Object → Trigger` (意味的に正しい方向) に修正、回帰防止テスト追加

### OSS Release Prep (v0.3.0)

- README に CI / License / Node / Tests バッジ追加
- Quickstart セクションを README トップに追加
- `.gitignore` に内部運用記録ディレクトリの除外を追加
- GitHub リポジトリ description / topics を設定
- claude-plugin/plugin.json の homepage URL を修正

## Getting Started

```bash
npm install -g @yohakuforce/core
yohaku --version
```

または `npx -p @yohakuforce/core yohaku init --bootstrap --profile minimal` で 1 回だけ試すこともできる。

コントリビュータ向けのソースからのビルドは [`CONTRIBUTING.md`](../../CONTRIBUTING.md) を参照。

詳細: [docs/01-getting-started/quickstart.md](../01-getting-started/quickstart.md)

## Known Limitations (v0.4.0 で対処予定)

- `method-summary-table` がインライン SOQL `[SELECT ...]` を検出しない

- `/yohaku-explain` の一括実行モード (`--all` / カテゴリ単位) 未実装
