## Q1. 自組織のセキュリティガイドライン的に問題ないか？

**A.**
このツールは、`force-app/` 配下のメタデータ（クラス定義・フロー構造・権限設定など）を処理します。顧客データ（レコードのデータ）は一切扱いません。

セキュリティ設計として以下が実装されています。

- `secrets-rules.yaml` による機密フィールドのマスキング（ValidationRule の errorMessage 等）
- AI への生 XML 直接送信の禁止（知識グラフ経由のみ）
- パストラバーサル防止、SQL インジェクション対策

自組織のガイドラインへの適合可否については、具体的なガイドライン内容に照らして各自ご確認ください。懸念点があればセキュリティ担当 (`<セキュリティ担当 / 窓口>`) にご相談ください。

## Q2. 顧客プロジェクトに使えるか？

**A.**
技術的には使えます。ただし、以下の点を確認した上で判断してください。

- **顧客の同意**: AI ツールを使用することを顧客に説明・合意を得ているか
- **データの取り扱い**: AI に送信される情報の範囲（メタデータの構造のみ）を顧客に説明できるか
- **成果物の扱い**: 自動生成されたドキュメントの利用・保管方法

推奨手順は、まず社内検証プロジェクトで試行することです。実績を確認した上で、顧客プロジェクトへの適用可否を判断してください。

## Q3. 学習コストはどれくらいか？

**A.**
最初の動作確認まで 30 分、基本操作の習得まで 1〜2 時間を想定しています。

主要コマンドは 4〜5 個です。

| コマンド | 用途 |
|---|---|
| `yohaku init --bootstrap` | プロジェクトへの初期設定 |
| `yohaku graph build` | 知識グラフの構築 |
| `yohaku render` | ドキュメント生成 |
| `yohaku sync` | グラフ更新 + ドキュメント生成を一括実行 |
| `yohaku diff` | Git 差分の分類 |

Claude Code の操作に慣れていない場合は追加で 1〜2 時間かかります。

## Q4. 既存のオンボーディング資料と何が違うか？

**A.**
既存のオンボーディング資料（Word / Confluence / PDF 等）との主な違いは 2 点です。

**1. 常に最新の状態を保てる**
`yohaku sync` を実行するたびに `force-app/` の現状からドキュメントが再生成されます。手動更新が不要です。

**2. 対話形式で質問できる**
静的な資料は「全部読んでから理解する」形式ですが、`/onboard` は「聞きたいことを聞く」形式です。「この Flow はどういう条件で動くか」「このクラスに依存しているものは何か」など、自然語で質問できます。

既存資料を置き換えるものではなく、「運用の手間を下げながら最新性を維持する仕組み」として捉えてください。

## Q5. AI が間違えたら誰が責任を取るか？

**A.**
AI が生成した情報はすべて「参考情報」です。最終的な判断と責任は、それを使用した人間（開発メンバー）にあります。

この設計方針はツール自体にも明示されています。

- **決定的処理層（CLI）**: 再現性があるため検証可能
- **AI エージェント層（Claude）**: 出力には誤りが含まれる前提で使う。特にリリース判断・顧客説明・設計変更は必ず人間が確認する
- **人手補完層（開発メンバー）**: 業務意図の判断・最終承認は AI に委ねない

AI の出力を鵜呑みにしないことが前提の設計です。「AI が確認したから大丈夫」ではなく「AI の出力を参考に人間が確認した」という使い方が想定されています。

## Q6. 自分のプロジェクトに導入するにはどうすればよいか？

**A.**
以下の手順で導入できます。

1. **npm から CLI をインストール**: `npm install -g @yohakuforce/core`
2. 導入先プロジェクトのルートで `yohaku init --bootstrap` を実行する
3. `force-app/` の場所を設定ファイルに指定する
4. `yohaku graph build` で知識グラフを構築する
5. `yohaku render` でドキュメント生成を確認する

`yohaku init --bootstrap` は以下を一括生成します。

- `.yohaku/` 設定ディレクトリ
- `CLAUDE.md`（Claude Code への指示書）
- `docs/generated/`（ドキュメント出力先）

導入を検討する場合は、まずリポジトリ同梱の `examples/sample-project/` で動作確認してからプロジェクトへ適用してください。

## Q7. Windows で動作するのか？Mac だけか？

**A.**
Windows でも Mac / Linux でも動作します。Windows のパス区切り（バックスラッシュ）の互換性も確保されています。

```powershell
yohaku --version
```

セットアップ手順は `02-quickstart-windows.md` に記載しています。

## Q8. Claude Code の利用料はどうなるか？

**A.**
Claude Code は Anthropic の有料サービスで、API 利用量に応じた従量課金です。

ツールには `yohaku metrics record/show` コマンドでコスト計測する機能があります。実際の利用量を把握した上で、チームでの利用方法を検討してください。

通常の開発業務での利用であれば、`/onboard` 1 回あたり数円〜十数円程度が目安です（モデルとプロジェクト規模によって異なります）。

## Q10. このツールで Salesforce のコードを自動生成したり、本番デプロイできるか？

**A.**
このツールの目的にそれらは含まれていません。

- **コード自動生成**: 既存のコーディングエージェントの領分です
- **本番デプロイ**: `sf CLI` の領分です

yohakuforce が解くのは「属人化の解消」「運用タスクの自動化」「AI 駆動開発の再現性」の 3 点です。既存ツールとの二重化や置き換えではなく、補完的な位置づけです。
