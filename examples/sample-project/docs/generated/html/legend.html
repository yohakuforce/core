凡例 この設計書の読み方

フローチャート・処理ステップ・項目値の割り当て・色分けの意味をまとめています。各ページ右上の「凡例」から戻れます。

処理フロー図（フローチャート）のノード

Apex / トリガの「内部処理フロー」で使う図形の意味です。図形の形と色で処理の種類を表します。

図形意味説明
開始・終了メソッド（処理）の入口と出口。角丸の囲み。
データ取得（SOQL）データベースからレコードを取得する。平行四辺形。
データ操作（DML）レコードを登録・更新・削除する。平行四辺形。
分岐（条件判定）条件を満たすか（はい / いいえ）で処理が分かれる。ひし形。
繰り返し（ループ）コレクションを 1 件ずつ処理する。点線が繰り返しの戻り。
例外処理（try / 捕捉 / 後処理）エラーを捕捉して処理する。点線が例外発生時の経路。二重枠。
戻り値・例外送出呼び出し元へ値を返す、またはエラーを送出して終了する。円。

矢印のラベル「はい / いいえ」は分岐の結果、「繰返」は繰り返しの戻り、「例外発生」は例外時の経路を表します。図が読みにくい場合は各メソッドの「ツリー」表示に切り替えると入れ子構造で確認できます。

処理ステップ（テキスト版）のバッジ

「処理詳細」のメソッド別ステップで使うバッジです。

  • 入力 メソッドの引数（受け取る値）。
  • SOQL レコード取得。
  • DML レコードの登録/更新/削除。
  • 分岐 条件による分かれ道。
  • 繰り返し ループ処理。
  • 例外処理 try / catch。
  • 戻り値 呼び出し元へ返す値。
  • 例外送出 エラーを投げて中断。
  • 処理 その他の処理（代入・生成など）。

項目値の割り当て（値の決まり方）

区分説明
数式で自動計算数式項目。他項目から自動算出（入力不可）。
参照 (lookup)他オブジェクトへの参照（Lookup / 主従）。
選択リスト決められた選択肢から選ぶ。
初期値あり新規作成時に既定値が入る。
積み上げ集計 (自動)子レコードからの集計値（入力不可）。
処理で設定Apex / トリガ / フロー が値を設定する。
ユーザー入力画面・API からの手入力。

色分け（事実 と 解釈）

この設計書は「決定的に抽出した事実」と「AI による解釈・確認」を分けて表示します。

決定的（事実）: 通常の表・カード。ソースから機械的に抽出した内容で、再生成しても同じ結果になります。

AI 解釈・確認ブロック

青い枠は AI が業務的意味・項目設定の詳細・抜け漏れ確認などを補ったブロックです。人手で編集でき、再生成しても保持されます。
